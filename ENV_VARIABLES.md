# Environment Variables Configuration

## Required Environment Variables

### Microsoft Azure AD (Required for Email Access)
```
TENANT_ID=your-tenant-id-here
CLIENT_ID=your-client-id-here
CLIENT_SECRET=your-client-secret-here
```

### OpenAI API (Required for Email Analysis)
```
OPENAI_API_KEY=your-openai-api-key-here
```

### User Configuration (Required)
```
USER_EMAILS=user1@domain.com,user2@domain.com
```
For multiple users, separate with commas.

## Auto-Configured on Render (No Action Needed)

These are automatically set by Render:

```
DATABASE_URL=postgresql://...  # Set by Render from database
WEBHOOK_API_URL=https://...    # Set by Render from web service
WEBHOOK_API_KEY=...            # Auto-generated by Render
WEBHOOK_HOST=0.0.0.0
WEBHOOK_PORT=10000
USE_EXCLUSION_API=true
AUTO_CLEANUP_DAYS=14
```

## Optional for Local Development

```
EXCLUSIONS_DB_PATH=excluded_instances.db  # SQLite file path for local dev
WEBHOOK_HOST=127.0.0.1                    # Local API host
WEBHOOK_PORT=5000                         # Local API port
USE_EXCLUSION_API=false                   # Use direct DB instead of API
```

## How to Set Variables on Render

### Option 1: Through Dashboard (Recommended for secrets)
1. Go to your service in Render Dashboard
2. Navigate to "Environment" tab
3. Add each variable with its value
4. Click "Save Changes"

### Option 2: Through render.yml (For non-secrets)
Already configured in `render.yml` for auto-configured variables.

## Security Notes

- **Never commit** `.env` files with real credentials
- Use Render's "Generate Value" for API keys
- Keep `CLIENT_SECRET` and `OPENAI_API_KEY` secure
- The `.env` file is in `.gitignore` to prevent accidental commits
